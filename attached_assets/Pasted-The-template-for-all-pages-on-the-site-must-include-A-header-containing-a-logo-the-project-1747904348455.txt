
**Шаблон для всех страниц сайта должен включать:**  
- **Шапку** с логотипом — названием проекта, которое вы придумаете.  
- **Подвал** с полным именем студента и номером учебной группы.  
- **Левое боковое меню** со ссылками на все страницы сайта и их названиями.  

### **Главная страница**  
Должна содержать:  
- Заголовок.  
- Описание профессии (текст не менее 2000 символов).  
- Тематические изображения по профессии (минимум 1 изображение).  

### **Страница "Общая статистика"**  
Должна содержать следующие табличные данные и графики (вся информация должна быть представлена одновременно в таблицах и графиках). Все таблицы и графики должны иметь соответствующие названия.  
1. **Динамика уровня зарплат по годам** (учитываются все вакансии, даже с зарплатами в других валютах. Средний диапазон зарплат должен быть переведён в рубли по курсу ЦБ на первое число месяца публикации вакансии. Исключить вакансии с зарплатой выше 10 млн руб., так как они считаются ошибочно заполненными.)  
2. **Динамика количества вакансий по годам** (учитываются все вакансии, даже с пустыми полями).  
3. **Уровень зарплат по городам** (в порядке убывания; если вакансия указана в иностранной валюте, применяется тот же пересчёт, что и для динамики зарплат по годам).  
4. **Доля вакансий по городам** (в порядке убывания; учитываются все вакансии, даже с пустыми полями).  
5. **ТОП-20 навыков по годам** (наиболее часто встречающиеся навыки, на основе анализа поля `key_skills`).  

### **Страница "Востребованность"**  
Должна содержать следующие табличные данные и графики (вся информация должна быть представлена одновременно в таблицах и графиках). Все таблицы и графики должны иметь соответствующие названия.  
1. **Динамика уровня зарплат по годам для выбранной профессии** (требования такие же, как для общей динамики зарплат по годам).  
2. **Динамика количества вакансий по годам для выбранной профессии** (требования такие же, как для общей динамики количества вакансий по годам).  

### **Страница "География"**  
Должна содержать следующие табличные данные и графики (вся информация должна быть представлена одновременно в таблицах и графиках). Все таблицы и графики должны иметь соответствующие названия.  
1. **Уровень зарплат по городам для выбранной профессии** (в порядке убывания; требования такие же, как для динамики зарплат по годам).  
2. **Доля вакансий по городам для выбранной профессии** (в порядке убывания; требования такие же, как для динамики количества вакансий по годам).  

### **Страница "Навыки"**  
Должна содержать следующие табличные данные и графики. Все таблицы и графики должны иметь соответствующие названия.  
- **ТОП-20 навыков по годам для выбранной профессии** (наиболее часто встречающиеся навыки, на основе анализа поля `key_skills`).  

### **Страница "Последние вакансии"**  
При посещении страницы система должна получать **IT-вакансии**, соответствующие выбранной профессии, **из HH API в реальном времени** (за последние 24 часа, максимум 10 вакансий, отсортированных по дате публикации). Каждый элемент списка должен содержать следующие поля:  
- Название вакансии.  
- Описание вакансии (*усложнённое задание, требует дополнительного GET-запроса*).  
- Навыки (перечислены в одну строку через запятую) (*усложнённое задание, требует дополнительного GET-запроса*).  
- Название компании.  
- Зарплата.  
- Название региона.  
- Дата публикации вакансии.
